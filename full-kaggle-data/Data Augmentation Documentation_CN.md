# 鸟类发声：全量数据整理、长尾平衡与音频增强说明

**版本 2.0 | 更新日期：2026-07-18**

## 1. 核心方案

正式方案保留清洗后的全部有效鸟类录音。我们不再通过删除高频鸟种录音来处理长尾，而是在训练阶段使用：

1. **平衡采样**：每个鸟种总体被抽到的概率相同；
2. **温和类别加权**：长尾类的错误在 loss 中更重要，但权重有上限；
3. **定向音频增强**：只增强低资源鸟种，且只在每个 CV 折的训练部分增强。

这能保留高频鸟种在录音人、地点、季节、设备和背景噪声上的真实多样性，同时避免它们占满 batch 或主导优化。

## 2. 数据来源、清洗与正式划分

输入文件为 `metadata/ml_full_deduped.csv`，包含 167,308 条字段级去重记录。

| 清洗规则 | 原因 |
| --- | --- |
| 删除纯数字 `primary_label` | 这类标签为非鸟类；共排除 101 个非鸟类物种。 |
| 删除 `rating <= 0.5` | 排除未评分和极低质量音频。 |
| 按 `rating` 生成 SNR 档位 | High：>=4.0；Medium：2.0-3.9；Heavy：<2.0。 |

清洗后得到 **144,250 条鸟类录音、1,126 个鸟种**。使用随机种子 42 固定划分：

| 数据集 | 条数 | 鸟种数 | 用途 |
| --- | ---: | ---: | --- |
| 清洗后鸟类总数据 | 144,250 | 1,126 | 总数据来源 |
| 训练池 | **129,834** | **1,126** | 训练与交叉验证 |
| 固定留出测试集 | **14,416** | 1,093 | 最终干净与加噪评估 |

总样本少于 5 条的鸟种全部保留在训练池，避免抽走其唯一训练样本。其他鸟种约 10% 的合格记录进入测试集，并对齐到 10% SNR 测试配额。

| SNR 档位 | 训练条数 | 测试条数 |
| --- | ---: | ---: |
| High | 86,731 | 9,631 |
| Medium | 41,011 | 4,553 |
| Heavy | 2,092 | 232 |

## 3. 为什么要用全量训练池

原有按物种封顶的版本有 99,960 条训练样本。它能限制不平衡，但同时会丢弃约 3 万条有效录音；这些录音包含同一鸟种在不同地点、时间和录制条件下的变化，对鲁棒性有价值。

正式思路是：

```text
保留全部有效鸟类数据 -> 固定留出测试集 -> 剩余全部训练
                                  -> 训练时通过采样器和 loss 控制长尾
```

这并不表示高频类可以主导训练。采样器控制每类进入 batch 的频率，类别权重控制每类错误对 loss 的影响。

## 4. 训练时如何处理长尾不平衡

### 4.1 平衡采样

`02_train_full_weighted.csv` 为每一行提供：

```text
sampler_weight = 1 / 该鸟种在训练集中的样本数
```

同一鸟种所有样本的采样权重之和为 1。因此将该列传给加权采样器（例如 PyTorch 的 `WeightedRandomSampler`）后，每个鸟种总体的抽样质量相同。它只控制“抽取样本的频率”，不是 loss 权重。

### 4.2 温和类别加权

同一 CSV 还提供 `loss_class_weight`：

```text
原始权重(c) = 1 / sqrt(该类别训练样本数)
最终权重(c) = clip(原始权重(c) / 原始权重均值, 0.25, 4.00)
```

我们刻意不用 `1 / 样本数` 作为 loss 权重，因为极少样本类会产生过大的梯度。逆平方根更温和；再限制到 **0.25-4.00**，能帮助长尾类，同时保证训练稳定。将它按类别索引传入分类损失函数即可；不要与 `sampler_weight` 相乘。

| 机制 | 解决的问题 | 当前实现 |
| --- | --- | --- |
| 全量训练池 | 保留头部类的声学多样性 | 129,834 条原始训练录音 |
| 平衡采样 | 防止头部类占满 batch | 每条样本权重为 `1 / 类别样本数` |
| 温和加权 loss | 放大尾部类错误的作用 | 逆平方根权重，限制 0.25-4.00 |
| 音频增强 | 补充最少样本类别的变化 | 仅样本数 <15 的训练鸟种 |

除总体 Top-1 accuracy 外，应同时报告 macro-F1、balanced accuracy、各 SNR 档位表现，以及按类别样本量分组的表现。

## 5. 低资源鸟种的音频增强

只对训练集中少于 **15 条**的鸟种增强，目标是达到至少 15 个有效训练样本。验证集和测试集始终使用原始音频。

| 增强方法 | 参数 | 模拟的现实变化 |
| --- | --- | --- |
| 时间拉伸 | 0.85x、0.90x、0.95x、1.05x、1.10x | 鸣叫节奏变化 |
| 音高偏移 | -2、-1、+1、+2 半音 | 个体/地域音高差异 |
| 加噪 | 5、10、15 dB SNR | 不同录音环境 |
| 音量增益 | 0.70x、0.85x、1.15x、1.30x | 距离和设备增益差异 |

每条原始音频最多有 16 个候选增强版本；只生成补足目标所需的数量，每个鸟种最多新增 50 条，避免人工样本过度占比。

YAMNet 端到端训练建议在线增强：

```text
建立某一 CV 折波形缓存
-> 找出该折训练集中样本数 <15 的鸟种
-> 仅生成训练样本的增强波形
-> 继承原始标签并加入训练缓存
-> 使用平衡采样 + 温和类别加权训练
```

增强发生在折内划分后，因此同一原始录音的增强版本不会进入验证集，避免数据泄漏。

## 7. 复现与评估规范

生成命令：

```bash
cd E:\MLwork\MLwork\data\augmenteddata
python prepare_full_training_data.py
```

脚本从 `metadata/ml_full_deduped.csv` 读取数据并固定使用随机种子 42。严格 5 折交叉验证时，建议依据每一折的训练 CSV 重新计算类别权重，保证验证数据不参与该折权重估计。

模型选择只使用 CV；最终评估前不要读取或使用 `03_test_holdout.csv`。最终先评估干净测试音频，再用固定噪声种子对同一测试音频生成 5 dB、0 dB、-5 dB 条件。LightGBM、FastAI CNN 和 YAMNet 统一使用这些测试条件，结果即可公平比较。
